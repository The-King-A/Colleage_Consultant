# 用户使用指南

本文说明从零到完成一次志愿填报辅助的**完整流程**（含环境准备、启动顺序与页面操作）。

---

## 一、环境准备

### 1.1 必备软件

| 软件 | 建议版本 | 用途 |
|------|----------|------|
| JDK | 17+ | Java 后端 |
| Maven | 3.8+ | 构建 Backend |
| Node.js | 18+ | 前端、Node 报告服务 |
| MySQL | 8.0+ | 业务数据库 |
| Redis | 6+ | 会话记忆、向量（智能问答需要） |
| Python | 3.10+ | 冲稳保、线差洞察（可选但推荐） |

### 1.2 创建数据库

```bash
mysql -u root -p
```

```sql
CREATE DATABASE IF NOT EXISTS volunteer_assistant
  DEFAULT CHARACTER SET utf8mb4
  DEFAULT COLLATE utf8mb4_unicode_ci;
```

### 1.3 导入表结构与数据

在仓库根目录执行（路径按本机调整）：
导入sql文件


### 1.4 修改配置

编辑 `Backend/src/main/resources/application.yaml`：

- `spring.datasource`：MySQL 地址、用户名、密码   写你自己的mysql账号和密码
- `spring.data.redis` / `langchain4j.community.redis`：Redis 地址
- `langchain4j.open-ai`：DeepSeek API Key（智能问答、推荐、评审）
- `langchain4j.open-ai.embedding-model`：SiliconFlow Key（RAG）
- `tavily.api-key`：可选，用于聊天联网搜索

Python 分析服务（可选）：

```bash
cd Services/analytics-service
copy .env.example .env    # Windows
# 编辑 .env 中的 DB_PASSWORD 等与 Java 一致
```

> **注意**：`consultant.data.run-sql-on-startup` 等开关保持 `false`，避免每次启动自动写库变慢。

## 二、启动服务（推荐顺序）

### 步骤 1：启动 Redis 与 MySQL

确保本机 Redis、MySQL 已运行。

### 步骤 2：启动 Java 后端（必须）

```bash
cd Backend
mvn spring-boot:run
```

成功标志：控制台无报错，浏览器访问 <http://localhost:8080/api/schools/page?page=1&size=1> 有 JSON 返回。

### 步骤 3：启动前端（必须）

```bash
cd Front/Consultant
npm install
npm run dev
```

浏览器打开：**<http://localhost:5173>**

### 步骤 4：启动 Python 分析服务（冲稳保 / 线差洞察需要）

```bash
cd Services/analytics-service
pip install -r requirements.txt
python main.py
```

成功标志：冲稳保页右上角显示 **「分析服务在线」**。

### 步骤 5：启动 Node 报告服务（导出 HTML 需要）

```bash
cd Services/report-service
npm install
npm start
```

未启动时，冲稳保「导出简报」等按钮会不可用，不影响测算本身。

## 三、推荐使用流程（考生一次完整填报）

下面按**实际使用顺序**排列，可按需跳过步骤。

```mermaid
flowchart LR
  A[注册登录] --> B[兴趣测评]
  B --> C[位次换算]
  C --> D[冲稳保测算]
  D --> E[填报向导]
  E --> F[保存方案]
  F --> G[AI评审]
  G --> H[智能问答深化]
```

### 3.1 注册与登录

1. 打开 <http://localhost:5173>
2. 点击右上角 **登录** → **注册** 填写账号
3. 登录后可使用：志愿推荐、兴趣测评、填报向导、志愿方案、收藏等

### 3.2 了解自我：霍兰德兴趣测评

1. 顶部 **志愿工具** → **兴趣测评**
2. 完成题目并提交，查看 RIASEC 类型与 AI 解读
3. 需要报告时点击 **导出报告**（需 Node 报告服务）

### 3.3 明确成绩：位次换算

1. **志愿工具** → **位次换算**
2. 选择省份、科类、年份，输入高考分数
3. 查看对应 **累计位次** 与超越比例（数据来自 `score_segment` 表）

### 3.4 量化院校：冲稳保测算（核心）

1. 顶部 **冲稳保** 进入测算页
2. 设置：
   - **院校所在地**：想报的学校在哪个省（如河南）
   - **录取生源省**：你的高考省份（如河南）
   - **科类**：理科 / 文科
   - **高考分数**
3. 若结果院校很少或冲档为 0：
   - 先点 **补全样本数据**（同省组合一般数秒内完成）
   - 再点 **开始测算**
4. 查看 **冲 / 稳 / 保** 三档列表
5. 可选操作：
   - **导出简报**：Node 生成 HTML
   - **保存方案**：写入「我的志愿方案」（需登录）
   - **分数线洞察 →**：查看样本分布与线差原因

### 3.5 辅助决策：分数线洞察

1. **志愿工具** → **线差洞察**
2. 选择与冲稳保相同的省、科类、分数
3. 查看样本数、线差分布；可结合霍兰德代码看专业门类建议

### 3.6 查院校与地图

| 入口 | 操作 |
|------|------|
| **院校库** | 筛选省份、层次、关键词，进详情收藏 |
| **地图选校** | 志愿工具 → 地图选校，按省聚合标点 |
| **对比** | 多校多专业对比，可「问 AI」跳转聊天 |

院校/专业详情页的 **咨询 AI** 会自动带上问题进入智能问答。

### 3.7 五步填报向导

1. **志愿工具** → **填报向导**（需登录）
2. 按步骤填写：基本信息 → 成绩 → 意向地区 → 专业倾向 → 风险承受
3. 每步可 **保存**；顶部可展开已填步骤 **查看/修改**
4. 完成后可跳转冲稳保、志愿方案等页面

### 3.8 形成方案：保存与 AI 评审

1. 冲稳保测算后点 **保存方案**，或进入 **用户菜单 → 志愿方案**
2. 在方案列表查看历史方案
3. 选择方案执行 **AI 评审**（流式输出，需 DeepSeek 配置正确）

### 3.9 深化咨询：智能问答

1. **智能问答** 或首页 **开始咨询**
2. 可开启 **联网搜索** 查询最新政策
3. 支持多会话、侧边栏历史

### 3.10 AI 志愿推荐（补充）

**志愿工具** → **志愿推荐**：由 Java 大模型根据对话生成文字版建议，与冲稳保结构化结果互补，适合「不知道怎么填」时使用。

## 四、常见问题

### Q1：冲稳保显示「分析服务离线」

- 确认已执行 `python main.py` 且端口 8001 未被占用
- 确认前端通过 `npm run dev` 启动（代理 `/analytics`）

### Q2：补全样本数据超时

- 请 **重启 Java 后端** 使用最新逻辑（同省不再向 28 省扩展）
- 同省（如河南+河南）应在几秒内完成；跨省仅补一条链路

### Q3：智能问答无响应

- 检查 Redis 是否启动
- 检查 `application.yaml` 中 DeepSeek API Key 是否有效

### Q4：登录后仍跳转登录页

- 清除浏览器缓存或检查 token；确认后端 8080 正常

### Q5：院校数量为 0

- 执行 `import_china_undergraduate_schools.sql` 或参考 `sql/README_院校数据导入.md`

### Q6：位次换算无数据

- 执行 `score_segment.sql` 导入一分一段样本
